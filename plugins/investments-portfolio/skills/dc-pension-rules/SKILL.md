# DC형 퇴직연금 규제 준수 규칙

---

## 1. DC형 퇴직연금 핵심 제약조건

| 제약 | 내용 | 적용 |
|------|------|:----:|
| **위험자산 한도** | 최대 70% (비적격 위험자산 기준) | MUST |
| **위험자산 정의** | 주식형 + 주식혼합형 + 채권혼합형 모두 포함 | MUST |
| **안전자산** | 채권형, 예금, MMF만 해당 | OK |
| **적격TDF (면제 위험자산)** | 위험자산이나 70% 한도 분자에서 제외(riskExempt), 100% 편입 가능 | MUST |
| **ETF 직접투자** | 과학기술공제회에서 불가 (펀드만 선택 가능) | OK |
| **단일 펀드 한도** | 일반펀드 40% 초과 금지(자체 분산권고). 적격TDF는 40% 초과 시 WARNING(법적 ERROR 아님) | SHOULD |

### 1.1 자산 3-상태 분류 (CRITICAL)

DC형 자산은 다음 **세 가지 상태**로 구분한다. 이 구분이 70% 한도 계산의 기준이다.

| 상태 | 정의 | 70% 분모 | 70% 분자 |
|:----:|:-----|:--------:|:--------:|
| **① 위험자산** | `riskAsset=true` AND NOT 적격TDF (주식형·혼합형·비적격TDF 등) | 포함(O) | 포함(O) |
| **② 안전자산** | `riskAsset=false` (채권형·예금·MMF) | 포함(O) | 제외(X) |
| **③ 적격TDF (riskExempt)** | 위험자산이지만 적격 요건 충족 → 한도 면제 위험자산 | 포함(O) | **제외(X)** |

> **핵심**: 적격TDF는 "안전자산"이 **아니다**. 본질은 위험자산이나, DC/IRP 위험자산 70% 한도의 **법적 예외(면제)**로 분자에서만 제외된다(분모에는 포함). 근거: 금융위 보도자료 fsc.go.kr/no010101/73294 — "적격TDF는 위험자산 70% 한도의 예외로 100% 편입 가능".
> **비적격TDF**는 이 예외를 적용받지 못하며 **일반 위험자산**으로 70% 한도에 산입된다.

## 2. 위험자산 분류표 (DC형 기준)

> **CRITICAL**: 채권혼합형도 위험자산으로 분류됩니다. 이를 놓치면 70% 한도를 초과할 수 있습니다.

| 펀드 유형 | 분류 | 70% 한도 적용 | 비고 |
|-----------|:----:|:------------:|------|
| **주식형** | 위험자산 | O | 국내 주식 투자 펀드 |
| **해외주식형** | 위험자산 | O | 해외 주식 투자 펀드 |
| **주식혼합형** | 위험자산 | O | 주식+채권 혼합 (주식 비중 높음) |
| **채권혼합형** | **위험자산** | **O** | ⚠️ "채권" 포함되어도 위험자산! |
| 채권형 | 안전자산 | X | 국내 채권 투자 펀드 |
| 해외채권형 | 안전자산 | X | 해외 채권 투자 펀드 |
| 예금/MMF | 안전자산 | X | 원금 보장형 상품 |
| **적격TDF** | **riskExempt (면제 위험)** | **O(분모) / X(분자)** | 적격 요건 충족 TDF — 70% 분자 제외, 분모 포함. **안전자산 아님** |
| 비적격TDF | 위험자산 | O | 적격 요건 미충족 TDF → 일반 위험자산으로 70% 산입 |
| 골드/원자재 등 기타 | 분류 확인 | - | `riskAsset` 필드로 개별 판정 |

> ⚠️ 실제 펀드 분류는 `fund_classification.json`의 `riskAsset` 필드로 확인하세요.
> ⚠️ **이전 버전 오류 정정**: 과거 "기타 | 안전자산 | TDF"는 잘못된 분류였습니다. TDF는 안전자산이 아니며, 적격TDF는 면제 위험자산(riskExempt), 비적격TDF는 일반 위험자산입니다.

### 2.1 위험자산 한도 수식 (nonExemptRiskWeight)

70% 한도는 **비면제 위험자산 비중(nonExemptRiskWeight)** 기준으로 판정한다. 적격TDF는 분자에서 제외하되 분모(총 포트폴리오)에는 포함한다.

```
nonExemptRiskWeight = Σ weight(riskAsset=true AND NOT 적격TDF) / total
                    ≤ 70%
```

수식의 의미를 prose로 풀면 다음과 같다:

- **분자(Σ weight, 한도 산입 대상)**: `riskAsset=true`이면서 적격TDF가 **아닌** 모든 펀드의 비중 합계. 즉 일반 위험자산(주식형·해외주식형·주식혼합형·채권혼합형) **+ 비적격TDF**의 비중을 모두 더한다.
- **분모(total)**: 적격TDF·안전자산을 포함한 **전체 포트폴리오 비중(100%)**. 적격TDF는 분모에 포함되므로, 적격TDF 비중이 커질수록 일반 위험자산이 차지할 수 있는 여유 한도가 늘어나지는 않는다(분모 고정 100% 기준).
- **판정**: `nonExemptRiskWeight ≤ 70%`이면 위험자산 한도 PASS. 적격TDF 비중은 이 분자에 더해지지 **않으므로**, 적격TDF를 100%까지 편입해도 위험자산 한도 위반이 아니다.

**계산 예시**:

```
포트폴리오:
- 적격TDF 2045          50%   → riskExempt: 분모 O, 분자 X
- 해외주식형             30%   → 위험자산: 분모 O, 분자 O
- 채권형                 20%   → 안전자산: 분모 O, 분자 X

nonExemptRiskWeight = (30%) / 100% = 30% ≤ 70%  → PASS
(적격TDF 50%는 분자에서 제외되어 위험자산 한도에 산입되지 않음)

만약 위 적격TDF가 '비적격'이라면:
nonExemptRiskWeight = (50% + 30%) / 100% = 80% > 70%  → FAIL (ERROR)
```

### 2.2 적격TDF 적격성 판정 규칙

TDF가 **적격TDF(riskExempt)**로 인정되어 70% 분자에서 제외되려면 적격 요건을 충족해야 한다.

| 단계 | 규칙 | 비고 |
|:----:|:-----|:-----|
| 1 | `tdfQualified` 필드 확인 — **기본값 true** | 명시적 false면 비적격 |
| 2 | 펀드명/분류에 **'적격'** 토큰 존재 확인 | '적격' 명시 시 적격 확정 |
| 3 | 수동 override 가능 | 운용사/감독 자료로 적격성 수동 지정 |
| 4 | 위 요건 미충족 → **비적격TDF** | 일반 위험자산으로 70% **분자 산입** |

```
적격성 판정 의사코드:
  isQualifiedTDF(fund):
    if fund.tdfQualified == false: return false      # 명시적 비적격
    if '적격' in fund.name or fund.category: return true
    if manualOverride(fund) is set: return manualOverride
    return fund.tdfQualified  # 기본 true (TDF로 식별된 경우)
```

> ⚠️ **비적격TDF를 적격으로 처리 금지**: `tdfQualified=false`이거나 적격 토큰이 없고 override도 없는 TDF는 반드시 일반 위험자산(70% 분자 산입)으로 취급한다.

### 2.3 TDF vintage → 투자자 연령 매핑

TDF는 목표연도(vintage, targetYear)로 글라이드패스가 정해진다. 투자자 적합성 검토 시 vintage를 연령에 매핑한다.

```
targetYear ≈ birthYear + 60      # 은퇴연령 60세 기본 가정
→ birthYear ≈ targetYear - 60
```

| 매핑 방향 | 수식 | 적용 |
|:----------|:-----|:-----|
| vintage → 출생연도 | `birthYear = targetYear - 60` | 은퇴연령 미입력 시 폴백 |
| 출생연도 → vintage | `targetYear = birthYear + 60` | 적합 TDF 추천 |
| 은퇴연령 입력 시 | `targetYear = birthYear + 은퇴연령` | 사용자 입력 우선 |

> 📌 은퇴연령이 명시되면 그 값을 사용하고, **미입력 시에만 60세 기본 폴백**을 적용한다.

### 분류 확인 방법

```
1. fund_classification.json에서 펀드명 검색
2. riskAsset 필드 확인
   - true = 위험자산 (70% 한도 적용)
   - false = 안전자산

예시 (형식 설명용):
{
  "[해외주식형 펀드명]": {
    "riskAsset": true,  // ← 위험자산
    "category": "해외주식형"
  },
  "[채권형 펀드명]": {
    "riskAsset": false,  // ← 안전자산
    "category": "채권형"
  }
}
```

> ⚠️ 위 예시는 형식 설명용. 실제 펀드 분류는 fund_classification.json에서 확인.

## 4. 분산투자 규칙

### 4.1 단일 펀드 집중 한도

> **중요**: 단일 펀드 40% 한도는 **법규가 아니라 본 시스템의 자체 분산권고**이다. 실제 DC/IRP 법적 집중한도는 펀드 단위가 아니라 **동일 법인 발행 증권 30% / 동일 계열사 합산 40%**에 적용되며(자본시장법·근로자퇴직급여보장법 시행령상 위험자산 운용방법 규제), **펀드 자체에는 40% 법적 상한이 적용되지 않는다**.[^1]

| 규칙 | 기준 | 성격 | 위반 시 |
|------|:----:|:----:|:-------:|
| **단일 펀드 한도 (일반펀드)** | ≤ 40% | 자체 분산권고 | **ERROR** (분산 미흡) |
| **단일 적격TDF 한도** | 권고 40% | 자체 분산권고 | **WARNING** (법적 ERROR 아님) |
| **단일 운용사 한도** | ≤ 60% | 자체 분산권고 | WARNING |
| **권장 운용사 수** | 2개 이상 | 자체 분산권고 | INFO |

> 📌 **적격TDF 단일 40% 초과 처리**: 적격TDF는 그 자체가 내부적으로 다자산 분산된 상품이므로, 단일 적격TDF 비중이 40%를 초과해도 **WARNING(자체 분산권고 차원)**으로만 표기하고 **법적 ERROR로 처리하지 않는다**. 반면 **일반펀드는 40% 초과 시 종전대로 ERROR**를 유지한다.
>
> ⚠️ 단, 단일 적격TDF 비중이 40%를 초과하더라도 위험자산 70% 한도(§2.1 nonExemptRiskWeight) 판정에는 영향이 없다(적격TDF는 분자 제외).

[^1]: 법적 근거 정리 — DC/IRP의 펀드 단위 40%는 본 시스템의 자체 분산권고일 뿐 법정 한도가 아니다. 법정 집중한도는 동일 법인 발행 증권 30%, 동일 계열사 합산 40%(펀드 미적용). 위험자산 70% 한도 및 적격TDF 예외(100% 편입 가능)는 §1·§2.1 참조(근거: 금융위 fsc.go.kr/no010101/73294).

### 4.2 섹터 집중 한도

| 구분 | 한도 | 예시 | 근거 |
|------|:----:|------|------|
| **단일 테마** | ≤ 30% | AI, 반도체, 로봇, 헬스케어 | 테마 사이클 리스크 |
| **관련 테마 합계** | ≤ 40% | 반도체+AI+로봇 (기술 섹터) | 상관관계 높음 |
| **단일 지역** | ≤ 50% | 미국, 중국, 한국 | 지역 리스크 분산 |
| **신흥국 합계** | ≤ 30% | 중국+인도+브라질+베트남 | 고변동성 |

### 4.3 집중도 체크리스트

- [ ] 단일 펀드 비중 40% 초과 여부 확인
- [ ] 단일 운용사 비중 60% 초과 여부 확인
- [ ] 반도체 + AI + 로봇 합계 ≤ 40%
- [ ] 단일 국가(미국 제외) 비중 ≤ 30%
- [ ] 신흥국 합계 ≤ 30%

## 5. 섹터 상관관계 매트릭스 (v1.1 신규)

> **목적**: 테마 펀드 간 숨겨진 상관관계를 정의하여 집중 리스크 탐지를 지원합니다.
> **소비자**: fund-portfolio, compliance-checker

### 5.1 상관관계 수준 정의

| 수준 | 상관계수 | 의미 | 집계 규칙 |
|:----:|:--------:|:-----|:----------|
| **High** | ≥ 0.7 | 함께 움직임 | 반드시 합산하여 한도 적용 |
| **Medium** | 0.4 ~ 0.7 | 부분 연동 | 합산 권장, 경고 수준 |
| **Low** | < 0.4 | 독립적 | 별도 관리 |

### 5.2 섹터 간 상관관계 테이블

| Sector A | Sector B | 상관관계 | 근거 | 합산? |
|:---------|:---------|:--------:|:-----|:-----:|
| **반도체** | **AI** | High | AI 학습/추론에 반도체 필수 | YES |
| **AI** | **로봇** | High | 휴머노이드 로봇 = AI + HW | YES |
| **반도체** | **로봇** | Medium | 간접 연결 (AI 통해) | YES |
| **반도체** | **데이터센터** | High | 데이터센터 = 반도체 수요처 | YES |
| 헬스케어 | 바이오 | High | 동일 산업 | YES |
| 배당 | 금융 | Medium | 금융주 = 배당 비중 높음 | WARN |
| 배당 | 채권 | Low | 독립적 자산 | NO |
| 에너지 | 원자재 | Medium | 부분 연동 | WARN |
| S&P500 | 나스닥 | High | 미국 대형주 중복 | YES |
| 국내주식 | KOSPI | High | 동일 시장 | YES |

### 5.3 Tech/AI 섹터 그룹 (집중 주의)

> **⚠️ CRITICAL**: 아래 키워드 포함 펀드는 **Tech/AI 그룹**으로 합산하여 40% 한도 적용

| 키워드 | 대표 펀드 유형 | 그룹 | 상관 |
|:-------|:--------------|:----:|:----:|
| 반도체 | 글로벌반도체, 시스템반도체 | Tech/AI | High |
| AI | AI, 인공지능 | Tech/AI | High |
| 로봇 | 휴머노이드, 로보틱스 | Tech/AI | High |
| 테크 | 코어테크, 빅테크 | Tech/AI | High |
| 데이터센터 | 클라우드, 인프라 | Tech/AI | High |
| 4차산업 | 4차산업혁명 | Tech/AI | High |

### 5.4 집계 규칙

```
Step 1: 포트폴리오 내 모든 펀드의 섹터 분류
Step 2: 상관관계 High 섹터끼리 그룹화
Step 3: 그룹별 비중 합산
Step 4: 합산 비중 > 40% → 경고
Step 5: 합산 비중 > 50% → 위험 (분산 필요)
```

### 5.5 예시: 집중 리스크 탐지

**포트폴리오**:
- 글로벌반도체 20%
- AI펀드 15%
- 휴머노이드로봇 10%
- 헬스케어 5%
- S&P500 20%
- 채권 30%

**집계 결과**:
- Tech/AI 그룹: 반도체(20%) + AI(15%) + 로봇(10%) = **45%** → ⚠️ 40% 초과
- 미국 노출: S&P500(20%) + (Tech 펀드의 미국 비중) = 높음

**권고**: 로봇 펀드 10% → 5%로 축소, 다른 섹터로 분산

## 6. 생애주기 자산배분 (Life-Cycle Allocation)

### 5.1 학술적 근거

| 연구 | 발견 |
|------|------|
| **SSA 시뮬레이션 (2010)** | 공격적 생애주기 전략(높은 주식 비중)이 평균 수익률 높음, 단 리스크도 증가 |
| **Harvard Pension Study** | 젊은 투자자 = 인적 자본(미래 소득) 많음 → 금융 자산에서 높은 위험 감수 가능 |
| **SOA Life-Cycle Fund (2024)** | 위험 회피도가 자산 배분의 핵심 결정 요인 |

### 5.2 연령대별 권장 배분 (DC형 70% 한도 적용)

| 나이 | 위험자산 | 안전자산 | 근거 |
|:----:|:--------:|:--------:|------|
| **30대** | **70%** (MAX) | 30% | 긴 투자 기간, 인적 자본 풍부 |
| **40대** | **70%** (MAX) | 30% | 여전히 충분한 회복 시간 |
| **50대** | 60% | 40% | 은퇴 준비, 점진적 축소 |
| **55-60** | 50% | 50% | 위험 축소, 안정성 강화 |
| **60-65** | 40% | 60% | 은퇴 직전, 원금 보존 |

## 7. 리밸런싱 원칙과 규율

### 6.1 리밸런싱의 근거

| 연구 | 발견 | 시사점 |
|------|------|--------|
| **Morningstar (2020)** | 리밸런싱은 위험 통제에 효과적 | 변동성 20-30% 감소 |
| **Kitces (2015)** | 연 1회 리밸런싱이 최적 | 너무 자주 = 비용 증가 |
| **Research Affiliates** | 5% 밴드 트리거가 효율적 | 시간 기반보다 효과적 |

### 6.2 권장 리밸런싱 규칙

```
┌─────────────────────────────────────────────────────────────────┐
│                    권장 리밸런싱 규칙                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 정기 리밸런싱: 매년 1월 (or 반기 1회)                          │
│     → 모든 자산을 목표 비중으로 조정                               │
│                                                                 │
│  2. 임계값 리밸런싱: 비중 ±5%p 초과 시                            │
│     → 해당 자산만 조정 (분기 점검)                                │
│                                                                 │
│  3. 긴급 리밸런싱: 시장 -20% 이상 하락 시                          │
│     → 계획 유지, 추가 매수 검토 (감정적 매도 금지)                  │
│                                                                 │
│  리밸런싱 방법:                                                  │
│  ┌─ 방법 A (매도-매수): 초과 비중 매도 → 부족 비중 매수            │
│  └─ 방법 B (납입 조정): 부족 비중 펀드에 납입 집중                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.3 펀드 교체 트리거

| 트리거 | 기준 | 검토 주기 | 조치 |
|--------|------|----------|------|
| **성과 부진** | 동일 카테고리 하위 25% (3년 연속) | 연 1회 | 대안 펀드 검토 |
| **비용 상승** | 총보수 0.5%p 이상 인상 | 변경 발생 시 | 비용 효율성 재평가 |
| **펀드 규모 축소** | 순자산 50억원 미만 | 분기 1회 | 청산 리스크 대비 |
| **운용 역량 변화** | 펀드매니저 교체, 합병 | 발생 시 | 6개월 성과 모니터링 |
| **더 나은 대안** | 동일 전략 + 비용 0.3%p 이상 낮음 | 연 1회 | 점진적 교체 검토 |

### 6.4 교체하지 말아야 할 상황

| 상황 | 이유 | 올바른 대응 |
|------|------|-----------|
| 단기 성과 부진 (1년 미만) | 단기 변동성은 정상 | 계획 유지 |
| 시장 전체 하락 | 개별 펀드 문제 아님 | 리밸런싱 검토 |
| 감정적 판단 (뉴스, 소문) | 편향된 의사결정 | 규칙 기반 판단 |
| "핫한 펀드" 추격 | 과거 성과 ≠ 미래 성과 | 장기 전략 유지 |
| 환율 변동에 따른 해외펀드 매도 | 환율 예측 불가 | 장기 관점 유지 |
