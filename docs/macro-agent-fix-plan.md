# 거시경제 에이전트 환각 수정 계획

**문서 버전**: v1.0  
**작성일**: 2026-01-12  
**상태**: 검토 대기

---

## 1. 문제 정의

### 1.1 현상

portfolio-coordinator 파이프라인의 거시경제 에이전트들이 실시간 데이터를 **허위 생성(hallucination)**하고 있음.

| 에이전트 | 보고값 | 실제값 | 오차율 |
|:---------|-------:|-------:|-------:|
| index-fetcher (S&P 500) | 5,966 | 6,966 | -14% |
| index-fetcher (KOSPI) | 2,586 | 4,586 | -44% |
| rate-analyst (Fed Rate) | 4.25% | 3.50% | +21% |

### 1.2 근본 원인

1. **검증 없는 생성**: 에이전트가 웹검색 없이 숫자를 생성
2. **출처 미명시**: 데이터 소스 URL이 프롬프트에 없음
3. **교차 검증 부재**: 다중 소스 확인 절차 없음

### 1.3 영향 범위

```
index-fetcher ──┐
                ├──► macro-synthesizer ──► 최종 보고서 (오염)
rate-analyst ───┘
```

- **직접 영향**: macro-synthesizer 출력 오염
- **간접 영향**: 포트폴리오 권고의 거시경제 근거 신뢰도 저하
- **영향 없음**: fund-portfolio, compliance-checker (로컬 JSON 기반)

## 2. 해결 목표

### 2.1 성공 기준

| 항목 | 기준 | 측정 방법 |
|:-----|:-----|:----------|
| 지수 정확도 | 실제값 대비 ±2% 이내 | FRED/Trading Economics 대조 |
| 금리 정확도 | 실제값 일치 | Fed 공식 발표 대조 |
| 출처 커버리지 | 100% 데이터에 출처 명시 | 출력 검증 |
| 교차 검증 | 2개 이상 소스 확인 | 에이전트 로그 확인 |

### 2.2 비목표 (Out of Scope)

- 실시간 API 연동 (MCP 서버 개발)
- 자동화된 데이터 파이프라인 구축
- 펀드 분석 에이전트 수정 (이미 정상 작동)

## 3. 해결 방안

### 3.1 아키텍처 변경

```
[현재]
index-fetcher ──► (LLM 지식 기반 추정) ──► 숫자 출력

[변경 후]
index-fetcher ──► (웹검색 필수) ──► (3개 소스 교차검증) ──► 숫자 출력
                       │
                       ▼
              [필수 데이터 소스]
              ├── FRED (fred.stlouisfed.org)
              ├── Trading Economics
              └── Yahoo Finance / Investing.com
```

### 3.2 에이전트별 수정 내용

#### 3.2.1 index-fetcher 수정

**현재 문제**: 지수 데이터를 추정으로 생성

**수정 방향**:
1. 웹검색 **필수화** (검색 없이 숫자 출력 금지)
2. 최소 3개 소스 교차 검증 강제
3. 출처 URL 및 조회일시 필수 포함

**필수 데이터 소스**:
| 지수 | 1차 소스 | 2차 소스 | 3차 소스 |
|:-----|:---------|:---------|:---------|
| S&P 500 | FRED (SP500) | Yahoo Finance (^GSPC) | Investing.com |
| KOSPI | Trading Economics | Yahoo Finance (^KS11) | 한국거래소 |
| NASDAQ | FRED (NASDAQCOM) | Yahoo Finance (^IXIC) | Investing.com |

#### 3.2.2 rate-analyst 수정

**현재 문제**: Fed 금리를 과거 데이터로 보고

**수정 방향**:
1. Fed 공식 발표 확인 필수
2. FOMC 일정 대비 최신 결정 반영
3. 한국은행 금통위 결정 확인

**필수 데이터 소스**:
| 항목 | 1차 소스 | 2차 소스 |
|:-----|:---------|:---------|
| Fed Funds Rate | federalreserve.gov | FRED (FEDFUNDS) |
| 한국 기준금리 | bok.or.kr | Trading Economics |
| US 10Y Treasury | FRED (DGS10) | Treasury.gov |

#### 3.2.3 macro-synthesizer 수정

**현재 문제**: 상위 에이전트 오류를 그대로 전파

**수정 방향**:
1. 입력 데이터 검증 단계 추가
2. 출처 누락 시 경고 및 재요청
3. 이상치 탐지 (전일 대비 ±10% 변동 시 재확인)

### 4.2 구현 상세

#### T1: index-fetcher 프롬프트 재설계

**파일**: `.claude/agents/portfolio-coordinator.md` (또는 별도 에이전트 정의)

**추가할 프롬프트 섹션**:

```markdown
## 데이터 수집 프로토콜

### 필수 규칙
1. **웹검색 없이 숫자 출력 금지**: 모든 지수/금리 데이터는 반드시 웹검색으로 확인
2. **3개 소스 교차검증**: 최소 3개 출처에서 동일 데이터 확인
3. **출처 명시 필수**: 모든 숫자에 [출처: URL, 조회일시] 포함

### 지수 데이터 수집 절차
1. 웹검색 쿼리 실행: "[지수명] current price today"
2. 3개 소스에서 값 확인
3. 값 불일치 시 (±2% 이상): 가장 신뢰도 높은 소스 채택 + 불일치 명시
4. 출력 형식:
   - S&P 500: X,XXX.XX [출처: Yahoo Finance, 2026-01-12 09:00 KST]
   - 교차검증: FRED X,XXX / Investing.com X,XXX

### 금지 행위
- ❌ "약 X,XXX 수준" 등 추정 표현
- ❌ 출처 없는 숫자 제시
- ❌ 단일 소스만으로 확정
```

#### T2: rate-analyst 프롬프트 재설계

**추가할 프롬프트 섹션**:

```markdown
## 금리 데이터 수집 프로토콜

### 필수 규칙
1. **공식 소스 우선**: 중앙은행 공식 발표 > 뉴스 > 추정
2. **FOMC/금통위 일정 확인**: 최근 결정일 명시 필수
3. **변경 이력 포함**: 직전 금리 → 현재 금리 (변경일)

### Fed 금리 수집 절차
1. 웹검색: "Federal Reserve interest rate current 2026"
2. federalreserve.gov 또는 FRED 확인
3. 출력 형식:
   - Fed Funds Rate: X.XX%-X.XX% [출처: Federal Reserve, FOMC 2026-XX-XX 결정]
   - 이전: X.XX% → 현재: X.XX% (XXXX-XX-XX 변경)

### 한국 기준금리 수집 절차
1. 웹검색: "한국은행 기준금리 현재"
2. bok.or.kr 금통위 결정 확인
3. 출력 형식:
   - 한국 기준금리: X.XX% [출처: 한국은행, 금통위 2026-XX-XX 결정]
```

#### T3: macro-synthesizer 검증 로직

**추가할 프롬프트 섹션**:

```markdown
## 입력 데이터 검증

### 검증 체크리스트
하위 에이전트 출력을 수신하면 다음을 확인:

1. **출처 존재 여부**
   - [ ] 모든 숫자에 [출처: ...] 태그 존재
   - [ ] 출처가 신뢰할 수 있는 기관 (FRED, 중앙은행, 거래소 등)

2. **데이터 신선도**
   - [ ] 조회일시가 7일 이내
   - [ ] FOMC/금통위 최신 결정 반영 여부

3. **이상치 탐지**
   - [ ] S&P 500: 4,000~8,000 범위 (2026년 기준)
   - [ ] KOSPI: 2,000~4,000 범위
   - [ ] Fed Rate: 0%~10% 범위
   - 범위 이탈 시: 재확인 요청

### 검증 실패 시 행동
- 출처 누락: "⚠️ [항목]의 출처가 누락되었습니다. 재확인 필요."
- 이상치 탐지: "⚠️ [항목]이 예상 범위를 벗어났습니다 (값: X, 범위: Y-Z). 교차검증 필요."
```

#### T4: macro-critic 검증 기준 강화

**추가할 검증 항목**:

```markdown
## 환각 탐지 강화 기준

### 필수 검증 항목
1. **지수 데이터 검증**
   - 웹검색으로 현재 지수 확인
   - 보고서 값과 ±5% 이상 차이 시: ❌ FAIL

2. **금리 데이터 검증**
   - 웹검색으로 현재 금리 확인
   - 보고서 값과 불일치 시: ❌ FAIL

3. **출처 검증**
   - 출처 URL 접근 가능 여부 (선택적)
   - 출처 기관 신뢰도 평가

### 검증 결과 보고 형식
| 항목 | 보고값 | 검증값 | 출처 | 결과 |
|:-----|-------:|-------:|:-----|:----:|
| S&P 500 | X,XXX | X,XXX | [URL] | ✅/❌ |
```

### 4.3 테스트 계획

#### T5: 통합 테스트

**테스트 시나리오**:

```
1. portfolio-coordinator 전체 파이프라인 실행
2. 각 에이전트 출력에서 다음 확인:
   - [ ] index-fetcher: 3개 소스 교차검증 수행 여부
   - [ ] rate-analyst: 공식 소스 인용 여부
   - [ ] macro-synthesizer: 검증 단계 실행 여부
   - [ ] macro-critic: 정확도 검증 통과 여부
3. 최종 출력의 지수/금리 값을 실제 값과 대조
```

**성공 기준**:
- 모든 지수: 실제값 대비 ±2% 이내
- 모든 금리: 실제값과 일치
- 모든 데이터: 출처 명시됨
